# Sources

Sources are adapters that load data into RAGKit. All sources implement the same interface and can be combined freely.

```python
agent.use(source_a)
agent.use(source_b)
agent.index()   # indexes all sources
```

---

## Built-in sources

### MarkdownSource

Loads `.md` files from a file or directory (recursive).

```python
from ragkit import MarkdownSource

# Single file
agent.use(MarkdownSource("./README.md"))

# All .md files in a folder (recursive)
agent.use(MarkdownSource("./docs"))
```

**Metadata added to each document:**

| Key | Value |
|---|---|
| `source` | Absolute file path |
| `type` | `"markdown"` |

---

### TxtSource

Loads `.txt` files.

```python
from ragkit import TxtSource

agent.use(TxtSource("./transcripts"))
```

---

### JsonSource

Loads `.json` files. Supports arrays and nested objects.

```python
from ragkit import JsonSource

# Use a specific key as the document content
agent.use(JsonSource("./articles.json", content_key="body"))

# Without content_key: entire object serialized as content
agent.use(JsonSource("./data.json"))
```

**Example JSON:**
```json
[
  { "body": "Article content here", "title": "My Article", "date": "2026-01-01" },
  { "body": "Another article", "title": "Second Article", "date": "2026-01-02" }
]
```

The `body` field becomes the document content; `title` and `date` go into metadata.

---

### CsvSource

Loads `.csv` files row by row.

```python
from ragkit import CsvSource

# Use a specific column as content; other columns go to metadata
agent.use(CsvSource("./faq.csv", content_column="answer"))

# Without content_column: all columns joined as "key: value | key: value"
agent.use(CsvSource("./products.csv"))
```

**Example CSV:**
```csv
question,answer,category
"How do I reset my password?","Go to Settings > Security.","account"
```

---

### RestSource

Fetches data from an HTTP API.

```python
from ragkit import RestSource

agent.use(RestSource(
    url="https://api.example.com/articles",
    content_key="content",          # JSON key to use as document content
    headers={"Authorization": "Bearer token"},
    params={"limit": "100"},
    timeout=30,                     # seconds
))
```

The endpoint must return either a JSON object or an array of objects.

**Error handling:** `RestSource` raises `RestSourceError` with a clear message on:

- Network failure
- HTTP 4xx/5xx response
- Non-JSON response body
- Timeout

---

### DirectorySource

Scans a directory and loads all supported file types.

```python
from ragkit import DirectorySource

# Default: loads .md, .txt, .json, .csv
agent.use(DirectorySource("./data"))

# Custom extensions
agent.use(DirectorySource("./data", extensions={".md", ".rst"}))
```

Silently skips files that cannot be decoded as UTF-8.

## Plugin sources

Community-maintained sources are distributed as pip packages:

```bash
pip install ragkit-pdf      # PDF files
pip install ragkit-docx     # Word documents
pip install ragkit-notion   # Notion databases
pip install ragkit-github   # GitHub repos
```

After installation, plugins are automatically discovered:

```python
from ragkit_pdf import PDFSource
agent.use(PDFSource("./documents"))
```

## The `Document` object

All sources produce `Document` instances:

```python
from ragkit.core.base import Document

doc = Document(
    content="The main text content.",
    metadata={"source": "file.md", "type": "markdown"},
    id=None,          # auto-generated by the pipeline
    embedding=None,   # filled during indexing
)
```
